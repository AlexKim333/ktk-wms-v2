# Project Rules

1. **안정성 최우선 (No Regression):** 기존에 잘 동작하던 기능(다국어 번역, 기존 로직 등)이 새로운 패치나 기능 추가로 인해 파괴되거나 오작동하는 일(Regression)이 없도록 철저히 검증해야 합니다. 하나를 고치기 위해 다른 하나를 희생해서는 안 됩니다.
2. **사이드 이펙트 사전 차단:** 로직을 수정할 때는 해당 파일뿐만 아니라, 그 로직과 연관된 다른 컴포넌트나 화면(예: ProductListView와 PosView 간의 연관성)에 미칠 사이드 이펙트를 항상 먼저 고려하고 함께 조치해야 합니다.
3. **상용화 수준의 신뢰성 유지:** 이 프로젝트는 상용화를 목표로 하고 있습니다. 테스트 단계라 하더라도 프로덕션(상용) 환경이라는 마음가짐으로 책임감 있고 신뢰성 있는 코딩 패치를 진행해야 합니다.
4. **기존 코드 수정 시 사전 승인 필수 (Approval First):** 어떤 식으로든 이미 작성되어 동작하고 있는 기존 코드나 뷰(View)를 수정해야 할 필요가 생긴다면, **절대 임의로 코딩을 진행하지 말고 먼저 사용자에게 "어떤 이유로, 어디를, 어떻게 수정할 것인지" 명확히 설명(보고)한 후 승인을 받고 나서 코딩을 진행해야 합니다.** 기존 기능의 파괴(Regression)를 막기 위한 최후의 방어선입니다.
