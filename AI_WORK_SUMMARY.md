# KTK WMS V2 - AI 작업 내역 및 컨텍스트 요약

본 문서는 다른 기기(예: 집)에서 작업을 이어나갈 때, AI가 이전까지의 작업 흐름과 현재 상태를 빠르게 파악할 수 있도록 작성된 요약본입니다.

## 🌟 최근 주요 완료 작업 (다국어 i18n 뼈대 구축)

가장 최근 진행된 **다국어 기능(Multilingual Support)** 작업의 완료 상태는 다음과 같습니다:
- **사용 기술**: 프론트엔드 내장형 번역(`vue-i18n`, Zero API Traffic 방식).
- **지원 언어 (4개 국어)**: 한국어(ko), 영어(en), 스페인어(es), 중국어(zh).
- **구현된 파일 및 로직**:
  - `src/i18n/index.js`에 설정 완료 (초기값은 `localStorage`의 `lady_polo_lang` 값 참조).
  - `src/i18n/locales/*.json` 4개 국어 번역 파일 뼈대 세팅.
  - 공용 언어 변경기 `src/components/LanguageSwitcher.vue` 생성.
- **화면 적용 (POC 완료)**:
  - **로그인 화면 (`LoginView.vue`)**: 모든 텍스트를 `$t()`로 교체하고 전환기 부착 완료.
  - **메인 레이아웃 사이드바 (`PosView.vue`)**: 사이드바 메뉴 텍스트를 모두 다국어로 연동 완료. 앱 내부에서도 언어가 즉각 번역되는 것을 확인했습니다.

## 📌 이전 주요 논의 및 아키텍처 맥락

1. **ERPNext(Frappe) vs Vue 번역 아키텍처 결정**: 동적 데이터와 에러 메시지는 백엔드(Frappe)에서 번역하고, 정적인 UI 텍스트(버튼, 라벨 등)는 트래픽 낭비를 막기 위해 전적으로 프론트엔드(`vue-i18n`)에서 처리하기로 확정했습니다.
2. **모바일 뷰 분리 논의**: 안드로이드/아이폰용 별도 화면(UI) 설계에 대해 논의된 바 있습니다.
3. **상품 UI 및 재고 로직 / 실시간 재고조사 / Node 아키텍처 재설계 / 사용자 권한 관리** 등의 기획 및 설계가 진행되었습니다.

## 🎯 다음 AI에게 전달하는 지시사항 (Prompt for the next session at Home)

집에서 이 프로젝트를 이어받은 AI 모델은 현재 소스 코드를 분석한 후 다음 **다국어 확장 미션**을 최우선으로 진행해 주세요:

1. **회원가입(Signup) 화면 다국어 선호도 연동**
   - 사용자 회원가입 시(아이디, 비번, 이메일 입력 화면) **"선호 언어(Preferred Language)"**를 4가지(ko, en, es, zh) 중 선택할 수 있는 드롭다운(UI)을 구현하세요.
   - 이 값이 Frappe 백엔드(User Doctype)로 올바르게 전송되도록 로직을 작성하세요.

2. **프로필 수정(Profile Edit) 화면 연동**
   - 로그인 후 자신의 프로필을 수정하는 창에서, 선호 언어를 다시 변경할 수 있는 옵션을 제공하세요.
   - 변경 즉시 프론트엔드의 `i18n.locale` 상태를 업데이트하고, DB(Frappe)에도 변경 사항을 반영하세요.

3. **내부 화면 점진적 다국어화**
   - 회원가입/프로필 기능이 완료되면, 나머지 주요 비즈니스 화면(상품 리스트, 재고 조정 등)의 하드코딩된 한글을 `$t('...')` 변수로 교체하고 JSON 사전을 채워나가는 작업을 진행하세요.
